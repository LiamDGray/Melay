What is it?

Being able to reply to your friends and family using the method we all communicate with, SMS, from anywhere. There are many text message forwarding services out there. Melay does it different because it is open source, encrypted, and most important can be served up on a server you own.

Melay (Message Relay) uses services that you can provision for free on your own such as firebase, heroku, azure, and others. Or you could host the service on the desktop at home!

So whenever you get a SMS message on your android phone, it can be viewed and answered on your PC, Mac, tablet, web browser, or even another android phone.

How does it work?

There are plenty of cloud providers out there.

[Images of mentioned cloud providers]

We make it easy to spin up your own personal version of Melay to run on any of the options provided here. When configuring the app on your phone, desktop, or tablet, the url to your own web service will be used. In addition to using cryptography, you own the message data passing from your phone to the internet.

Push notifications are sent out from my own push notification server. These messages are passed encrypted from your server to my push notification server. This is done because many services such as Apple Notification Service and Windows Notification Service expect an app's notifications to come from a specific URL or with a key. My experience as a dev is that it is a bad idea to put the key to an application in the source code itself. If you need your own push notification server, the code for all the clients is open source at github and you are welcome to compile your own.

How do I get started?

First, select your hosting solution. This can be a cloud provider or on your own machine. There are some caveats to each approach.
